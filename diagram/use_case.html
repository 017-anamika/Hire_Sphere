Diagram Rationale & Explanation

This final Use Case Diagram defines the complete functional scope required for a perfect application foundation. It introduces multiple actors and standard UML relationships to ensure clarity.

  • Actors:
    • User / Professional: The primary actor. They perform all core social networking activities.
    • Recruiter: A specialized user. They can do everything a User can, but also have extended capabilities like posting jobs and searching for candidates with advanced filters.
    • System Admin: A privileged actor responsible for system maintenance, like moderating content or managing users.
  • Use Cases (Core Functionality):
    • Manage Profile: Encompasses creating, editing, and updating all profile sections (experience, skills, etc.).
    • Manage Network: Includes sending, accepting, and removing connections.
    • Interact with Content: Covers liking and commenting on posts.
  • Relationships:
    • `<>` (Authenticate User): This is a powerful concept. It signifies that most other use cases (like posting content or applying for a job) *must* include the "Authenticate User" step. This tells the developer that authentication is a core, reusable piece of middleware that will be applied to many different API routes.
    • `<>` (Post a Job): This indicates optional functionality. A `Recruiter` might have the primary use case to "Search for People," but they can *extend* that with the "Post a Job" functionality if they choose. It separates core features from optional or specialized ones.
