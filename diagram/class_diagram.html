Design Rationale & Relationships

  • Inheritance: User and Company extend Member. This allows both to share common functionality like being searchable or creating content. Post and Comment extend Content.
  • Composition: User (1) *-- (1) Profile. A Profile is fundamentally part of a User and cannot exist without one. If a User is deleted, their Profile is also deleted. This is a strong "owns-a" relationship.
  • Aggregation: Profile (1) o-- (0..*) Experience, Education, Skill. A Profile "has-a" list of these items. They are part of the Profile, but their lifecycle is not strictly tied to the Profile. This is a weaker "has-a" relationship.
  • Association: Member (1) -- (0..*) Content. A Member (User or Company) can create many pieces of Content.
  • Association Class: User (2) -- (Connection). The `Connection` class holds attributes about the relationship between two Users, such as its `status` (pending/accepted).
  • Self-Association: Comment (1) -- (0..*) Comment. A Comment can be a reply to another Comment, creating nested threads.
